HTML/CSS

In 1980 a physicist named Tim Berners-Lee proposed and prototyped ENQUIRE, which was a system CERN researchers used to share documents. In 1989 the same man wrote a memo proposing an Internet-based hypertext system. He specified HTML and wrote the browser and server software in late 1990

Css was simply a grouping of text-based code that overrides an internet browser's formatting so the developer can give it the style he desires. CSS works in tandem with HTML to control multiple webpages and site's appearance so developers and designers don't have to spend so much time making every page look the same, which is impossible with several hundred webpages.

Html and CSS are quite good at

  • Easy to learn and understand
  • Well-interpreted by browsers
  • A forgiving language, if an element fails to parse it wont break the document (Does not apply to XHTML)
  • It's light, with high signal to noise ratio compared to other forms of communication.
  • Vast array of structural and aesthetic elements that can infer meaning and presentation
  • Free, no license fees, DRM free, open standard and transparent
  • Hugely popular and widely used
  • Multiple languages supported to assist
  • Can go anywhere a browser can, thus is multi-platformed

The bad things about it. But really theres no substitute for websites

  • Requires a browser
  • Open to author interpretation making authors the weakest link
  • Technical progress is slow
  • The HTML Lexicon is set in American English. Requires extra interpretation by non-English authors

  • Css has a few problems such as being foreign syntax wise, it's still simple and easy, but it is vastly different from other languages
  • Can get complicated and messy when implementing third party software such as Dreamweaver or FrontPage

When to use it

When you're creating a web page. HTML is used to set up a website with content and slight formating such as headings, paragraphs, and other such things. CSS along with JavaScript are used to add more visual effects to the webpage, making it look more appealing to the user.
