HTML/CSS

In 1980, a physicist named Tim Berners-Lee proposed and prototyped ENQUIRE, which was a system CERN researchers used to share documents. In 1989, the same man wrote a memo proposing an Internet-based hypertext system. He specified HTML and wrote the browser and server software in late 1990.

CSS was simply a grouping of text-based code that overrides an internet browser's formatting so the developer can give it the style he desires. CSS works in tandem with HTML to control multiple web pages and sites appearance so developers and designers don't have to spend so much time making every page look the same, which is impossible with several hundred web pages.

What HTML and CSS are quite good at

  • Easy to learn and understand
  • Well-interpreted by browsers
  • Forgiving, if an element fails to parse it won't break the document (does not apply to XHTML)
  • Vast array of structural and aesthetic elements that can infer meaning and presentation
  • Free, no license fees, DRM-free, open standard and transparent
  • Hugely popular and widely used
  • Can go anywhere a browser can, thus has cross-platform support

The bad things about HTML and CSS

  • Requires a browser
  • Technical progress is quite slow
  • The HTML Lexicon is set in American English, therefore requires extra interpretation by non-English authors
  • Foreign syntax (especially CSS), though simple and easy, but still vastly different from other languages
  • Can get complicated and messy when implementing third party software such as Dreamweaver or FrontPage

When HTML and CSS are used

Simply when you're creating websites. HTML is used to set up the web pages with content and slight formating such as headings, paragraphs, and other such things. CSS (along with JavaScript) are used to add more visual effects to the page, making it look more appealing to the users.
